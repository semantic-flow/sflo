metadata flow

A metadata flow contains system-related administrative and structural metadata for every mesh node, including the versioning data for each node's flows.

In the filesystem, it exists as a _node-metadata-flow folder in a node folder.

Mesh-specific metadata about a node's flows' version snapshots mostly lives here too, eliminating the need to keep separate metadata in the node component. Also may contain metadata about the assets folder.

Use of _node-handle in metadata flows

When metadata flows (or any system facet dataset) refer to mesh nodes, they'll usually be talking about "the-node-as-mesh-constituent", so they'll use the node's node handle identifier
