default snapshot

Versioned mode: _default/ is a materialized copy of the latest snapshot folder. Unversioned mode: _default/ is the one-and-only dataset de-versioned flow facet mode: _default is the current version of the dataset

The default snapshot provides the most recently published version of a flow's content without relying on symlinks or redirects. It serves as the "general purpose" source that users and external systems can reference, remaining unchanged during active development via the working snapshot.

If versioning is turned on and nobody has cleaned up old versions, the default snapshot matches the content of the latest versioned snapshot (e.g., 2025-11-24_0142_07_v3/) and remains identical to the working snapshot until new changes begin. During weaving, the working distribution becomes the new default snapshot. If versioning is turned on, the _working content becomes the working version.

This provides a reference point for citations and external links that want the latest information, while allowing ongoing development work to proceed safely in the _working dataset without disrupting users of the published data.
