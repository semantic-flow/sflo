root node

The node at the top of a repository file hierarchy may be referred to as the root node. It is the namespace base, even if it is not a namespace node.

Every other mesh resource in a mesh "lives under" the root folder, and its identifier is the folder name, which is usually the repo-name if it's being created as a mesh repo.

It is not treated or represented any differently than any other MeshNode, and it is not differentiated in metadata (hopefully?)
