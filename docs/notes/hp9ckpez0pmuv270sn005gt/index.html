node folder

Definition

A node folder is any folder that maps to a mesh node. Each node folder extends the namespace with its name and has a concept URL that refers to something.

  • What a “namespace” is: see Namespace
  • General node types and anatomy: see mesh node

Minimal requirements

  • Every node folder must contain:
    • _node-handle/
    • _meta-flow/

Node-specific flows (by type)

  • bare node: no additional flows
  • data node: requires _data-flow/

Distributions must live inside flow snapshot folders (e.g., _current/, _vN/). See node flow and flow snapshot.

Example

/my-node/                     # node folder → https://ex.org/my-node/
├── _node-handle/             # required
├── _meta-flow/               # required
└── _data-flow/               # required for data nodes
