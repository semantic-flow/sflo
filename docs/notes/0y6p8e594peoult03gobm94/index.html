metadata flow

A metadata flow contains system-related administrative and structural metadata for every MeshNode, including the versioning data for each node's flows.

Physically, it exists as a meta flow folder in a node folder.

Mesh-specific metadata about a node's version snapshot mostly lives here too, eliminating the need to keep separate metadata in the element.

Use of _handle in metadata flows

When metadata flows (or any System dataset) refer to mesh nodes, they'll usually be talking about "the-node-as-mesh-constituent", so they'll use the node's handle identifier
