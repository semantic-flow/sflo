The **working version** serves as a draft workspace for ongoing changes to a knop's [[mesh-resource.component.flow]]s
After a version-bumping weave, a working version starts identically to the default dataset but can be modified safely without affecting the stable current version. During weaving, _working content becomes the new default dataset and gets versionted as the latest version, while _working naturally remains ready for the next round of drafts.

This allows continuous development and version control commits without requiring immediate version bumps or disrupting users of the stable dataset.
